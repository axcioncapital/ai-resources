# Pipeline Review — /consult — 2026-05-29

## Summary

`/consult` is the operator's structural-judgment channel into the `system-owner` agent (Opus), covering Functions A (general consultation) and B (pre-change advisory) of the seven-function System Owner contract. Its design centre is correct: a thin orchestrator that classifies question shape, loads the routing baseline only when needed, and returns the agent's voice verbatim — no shell prose, no synthesis layer between operator and System Owner. The shape risk it carries today is a sustained, telemetry-confirmed leak: the agent's full advisory text flows back into the main session unbounded, violating the Subagent Contracts cap in `ai-resources/CLAUDE.md` and producing a ~2–4k-token drip that has now recurred across at least five consecutive sessions (friction-log 2026-05-25 → 2026-05-28+) without structural fix. A secondary risk is silent drift between the canonical agent at `ai-resources/.claude/agents/system-owner.md` and the project-local copy at `projects/axcion-ai-system-owner/.claude/agents/system-owner.md` — the project-local copy carries two extra v1-scoping sentences that are now factually incorrect (the agent is no longer project-local; it serves six commands across the workspace). The Step 0 read-first gate is `[FADING-GATE] verified 2026-05-27` and is doing its job. Currency check (per § 3 below) flags the unified Anthropic skills/commands convention — load-bearing for the v1.1 cycle but not a blocker.

## Innovations proposed

- **Cap the advisory return at the contract floor (sub-30-line summary + on-disk advisory).** — The `system-owner` agent at v1 has `Write` scoped to `projects/axcion-ai-system-owner/output/` already. Extending Function A and Function B to write the full advisory to `output/consultations/consult-{DATE}-{slug}.md` and return a ≤30-line summary to the main session brings `/consult` into structural conformity with `ai-resources/CLAUDE.md § Subagent Contracts` — which Functions C, E, F, and G already follow. This is the single highest-leverage change for `/consult` (cited in friction-log 2026-05-25, -26, -27, -28 as a recurring lever, last entry: "fixing it once collapses a chronic drip"). Affects: `.claude/commands/consult.md` Step 4 brief, `.claude/agents/system-owner.md` Phase 5 Function A/B output shape. Risk class: structural change class — gated by `/risk-check` per `audit-discipline.md`. Grounds: `principles.md § QS-9` (automation passes the same quality gates), `risk-topology.md § 3` (canonical-agent edit blast radius). Note: Function A's "free-form structured answer, target under 60 lines" sometimes converges below the cap, in which case the on-disk write is skipped — only the return-summary contract is mandatory.
- **Merge `/consult` and `/pm`'s change-shape definition into a single shared snippet** — the change-shape list lives verbatim in two places: `consult.md` Step 2 (lines 42-58) and `project-manager.md` Phase 3 (lines 63-79). The two-end-contract warning in `consult.md` § Step 2 acknowledges the brittleness and `risk-topology.md § 5` ("Change modifies a string literal matched by another component (two-end contract)") classifies it as a structural-risk signal. Extracting both to `ai-resources/docs/change-shape-definition.md` and read-including from both consumers eliminates the silent-drift class. Affects: `.claude/commands/consult.md`, `.claude/agents/project-manager.md`, new `docs/change-shape-definition.md`. Risk class: structural change class. Grounds: `principles.md § DR-7` (generalize only when a second confirmed consumer exists — here, both confirmed). Counter-argument the operator should weigh: per `principles.md § AP-7`, two consumers is the *minimum* threshold; the snippet is short and stable. Defer if friction-log shows no drift yet — surface only when one side actually changes without the other.

- **Currency check — Substantive: unified skills/commands convention.** Per the Anthropic platform docs at `https://code.claude.com/docs/en/skills` (fetched 2026-05-29), "Custom commands have been merged into skills. A file at `.claude/commands/deploy.md` and a skill at `.claude/skills/deploy/SKILL.md` both create `/deploy` and work the same way. Your existing `.claude/commands/` files keep working." The legacy `.claude/commands/consult.md` form continues to be supported — no functional break. Severity: Minor at the file level; Substantive at the system level — this is the convention shift that will eventually touch all 47 commands in the registry, and `/consult` is a reasonable candidate for the migration pilot given its small size and well-defined contract. Defer-or-do is the operator's call; logging here per the auditor's currency-check contract.
