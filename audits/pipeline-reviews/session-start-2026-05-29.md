# Pipeline Review — session-start — 2026-05-29

## Summary

`/session-start` is the Phase 3 mandate-capture orchestrator: 4 steps that read `logs/session-notes.md`, parse `$ARGUMENTS` or one operator answer into a structured mandate, echo a styled confirmation, and append a parse-contract-pinned `**Mandate:**` block back to `logs/session-notes.md`. The design center is sound — a single load-bearing two-end contract (`Step 3 bullet labels`) feeds three named readers (`wrap-session.md` 7a, workspace `wrap-session.md` 2b, `drift-check.md` 5), and the contract is documented in-line at both ends per `risk-topology.md § 1` two-end-contract guidance. Shape risk is concentrated in three places: (a) the Step 0.5 marker-file mtime guard, which carries documented residual TOCTOU risk and falls back to a 120s heuristic when the marker is absent; (b) the Step 2 confirmation parser, which was hardened on 2026-05-16 against single-letter ambiguous replies but still depends on operator discipline with the `b: <text>` correction syntax; (c) a stale reference to a non-existent `mandate-parser` skill in the boundary note. The command is current against the unified Anthropic skills/commands docs page, well-aligned with `principles.md § OP-3` (loud over silent — heuristic fallbacks emit visible `[Step 0.5] Note:` lines), and at 233 lines is dense but proportionate to the contracts it carries.

- **Step 0 / Step 3 duplicate `session-notes.md` read justification** — Step 0 reads the last 10 lines for the precondition check; Step 3 says "Using the `logs/session-notes.md` content already read in Step 0, locate today's session header. If Step 2 took longer than ~30s or a concurrent session may have written to `logs/session-notes.md` during Step 1/Step 2, re-read the last 10 lines before locating the header." This is the SF1 fix from commit `2965a21` (eliminate duplicate Step 0 read), but the re-read condition reopens the duplicate-read path on every session over the 30s threshold — which is essentially every session that uses operator confirmation. `usage-log.md` line 49 and 260 confirm `session-notes.md` is read 3–4× per session and flag this as a recurring lever. Severity: Substantive — the 2026-05-22 SF1 fix did not fully close the re-read, and the issue is recurring telemetry per the usage log.
- **Step 0.5 absent-file guard leaves a silent skip path** — Lines 38–39 ("if `SESSION_NOTES_MTIME` is empty ... Skip Step 0.5 entirely and proceed to Step 1. Per `principles.md § OP-3`, this is a silent continuation — the operator cannot tell from chat whether Step 0.5 ran clean (DELTA=0), ran with foreign-write detected, ran with heuristic fallback, or skipped entirely. Severity: Minor — the absent-file case is genuinely rare (fresh repo only), but the asymmetry violates the loud-fallback convention this command otherwise establishes.
