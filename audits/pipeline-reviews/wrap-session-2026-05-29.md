# Pipeline Review — wrap-session — 2026-05-29

## Summary

`/wrap-session` is the canonical session-close pipeline — it writes the session note, decisions, coaching profile, and usage telemetry, runs innovation/improvement triage, fires the end-time `/risk-check` gate, and commits. Its design center is *reliable shared-state writes under concurrent-session pressure* — most of its complexity (preflight, Step 0/0.5/3/3.5, classifier) exists to prevent silent clobber of `logs/session-notes.md`. The pipeline is heavily iterated (15+ recent commits) and currently carries Substantive shape risk in three places: (1) the embedded ~85-line foreign-session detector violates principles.md § DR-7 (specific-before-general) only nominally — it has earned its complexity through repeated incident — but it is now load-bearing inline Bash that drifts silently if the writer-side conventions in `prime.md` / `session-start.md` / `check-archive.sh` change, (2) the canonical and workspace-root copies are not symlinked and must be hand-synced via the PAIRED CONTRACT comment block, and (3) auto-mode chain false-positives (logged 2026-05-28) are formally deferred to id-31 Phase 2 but the pipeline does not state the live workaround. The currency check passes — the `model: sonnet` frontmatter aligns with the unified skills/commands docs current as of 2026-05-29.

## Innovations proposed

- **Move the Step 3.5 detector into a dedicated `scripts/foreign-session-guard.sh` script** — the 85 inline Bash lines plus paired awk classifier are the largest single block in the command body and create a structural sync hazard with the workspace-root copy. Externalizing the detector to a script that both copies invoke (`bash logs/scripts/foreign-session-guard.sh "$TODAY"`) removes the dual-maintenance burden and reduces the body to ~10 lines of "run the script; interpret JSON output". This is grounded in principles.md § DR-3 (component homes are fixed — scripts live in `ai-resources/scripts/`) and risk-topology.md § 1 High (two-end contract — string literals matched by another component). Affects: `.claude/commands/wrap-session.md` (canonical + workspace-root), new `logs/scripts/foreign-session-guard.sh`. Risk class: structural — new shared-state script qualifies for `/risk-check` per audit-discipline.md.
- **Add a `Mandate` echo-and-skip path for sessions that ran `/prime` but never `/session-start`** — Step 7a currently sets `mandate_present = false` when no mandate line exists, but does not record *why*. Adding a one-line classification (`no_session_start | aborted | trivial`) lets the coaching-data signal distinguish three meaningfully different states. Grounded in principles.md § OP-7 (session signals must be loud, not silent). Affects: Step 7a, `coaching-data.md` schema. Risk class: minor — schema additive.
- **Inline a one-line "auto-mode chain detected — guard may false-positive; verify HEAD" hint when `PRIME_RAN=1` but `ADDED_HEADERS > 1`** — the friction-log entry of 2026-05-28 documents that auto-mode chains (`N,M auto` from `/prime`) legitimately add multiple today-headers, and the deferred fix requires id-31 Phase 2 marker-scoping. Until that lands, the live workaround is operator override after manual HEAD audit. Surfacing the chain hint inline in the guard's STOP block converts a recurring drift event into a one-step resolution. Grounded in principles.md § OP-3 (loud over silent — name the known false-positive). Affects: Step 3.5 STOP message block. Risk class: minor — documentation-only addition.

## Leanness fixes

- **Collapse the preflight prompt into a single two-letter answer (default `yy`)** — the current preflight asks two y/n questions and a paragraph of accepted shorthand. The structural fix is to default to `yy` and require explicit opt-out (`n1`, `n2`, or `nn`), eliminating the two-tail re-ask path. Estimated token reduction: small (~50 tokens per session, ~30 lines of body removed). Risk class: minor.
- **Strip Step 11 (`/improve` reminder)** — the check runs a grep against today's date in friction-log.md and emits a suggestion. The same signal is available to the operator via `/session-start` Step 0.5 of the next session and via `/friday-checkup`. Three reminder paths for one signal violates principles.md § AP-4 (rubber-stamp approvals — the suggestion creates no decision, only acknowledgment). Estimated token reduction: small (~150 tokens, one bash call + 3 lines). Risk class: minor — removes only the in-command nudge; friction-log itself unchanged.
- **Drop the "shared command drift check" (Step 9)** — Step 9 asks the operator a question about syncing `.claude/commands/` edits to other projects, but auto-sync-shared.sh (SessionStart hook per risk-topology.md § 1 Critical) propagates canonical command changes to all projects automatically on next session. The question is asked at the wrong layer and at the wrong time — by wrap-session, the canonical edit is already in the index. Estimated token reduction: small (~120 tokens). Risk class: minor — Step 9 is advisory-only; removing it loses no enforcement.

## Brokenness / drift noted

- **PAIRED CONTRACT: workspace-root copy is hand-synced, not symlinked.** Evidence: `diff -q` between `/Users/patrik.lindeberg/Claude Code/Axcion AI Repo/.claude/commands/wrap-session.md` and the canonical `ai-resources/.claude/commands/wrap-session.md` reports the files differ. The PAIRED CONTRACT comment block at canonical line ~40 acknowledges this and lists symmetric requirements with `/session-start` Step 0.5. Severity: **Substantive**. This is exactly the two-end-contract drift class that risk-topology.md § 5 "Signals that elevate a change to structural risk" warns about — string literals matched by another component. The Innovation 1 proposal (externalize detector to a script) directly mitigates this.
- **Auto-mode chain false-positive in Step 3.5 guard — deferred but not surfaced in command body.** Evidence: friction-log.md line 86 (2026-05-28) documents that `/prime` Step 8c.3 fires N times under `N,M auto` chains, legitimately adding multiple today-headers; guard computes `FOREIGN = ADDED_HEADERS - PRIME_RAN` with `PRIME_RAN=1` and false-positives. Severity: **Substantive**. The fix is formally deferred to id-31 Phase 2 (marker-scoped session-notes headers), but until that ships, the wrap command's STOP message does not name the known false-positive class. Operators must remember the workaround.
- **Step 0.5 continuity scratchpad calls `skills/handoff/SKILL.md` inline — the handoff skill's `disable-model-invocation: true` frontmatter blocks model invocation, which is the correct posture, but the wrap-session body says "Run the handoff continuity workflow … inline" without specifying the invocation mechanism.** Evidence: wrap-session.md line 23; handoff SKILL.md frontmatter line 14. Severity: **Minor**. The current behavior works — the wrap command effectively executes Steps C1–C2 from memory of the skill body — but the indirection is documented ambiguously. State the invocation contract explicitly ("execute the C1 Write call directly, do not invoke `/handoff`").
- **Cost-budget threshold (25 tool calls) is structural but unverified.** Evidence: wrap-session.md line 13. The token-audit entries in usage-log.md repeatedly show wrap sessions hitting ~22 Bash calls inherent to the protocol (line 1139: "Tool overhead — Minor: Bash count (~22) inflated by /wrap-session protocol"). Severity: **Minor**. The 25-call ceiling is within 3 of the routine baseline; one additional friction-log grep or innovation-registry check pushes a clean wrap into the abort-prompt path. The threshold either needs raising to ~35 or the protocol needs the Leanness fixes above to create headroom.
- **Currency check passed.** WebFetch of `https://code.claude.com/docs/en/skills` (2026-05-29) confirms the unified skills/commands convention: files in `.claude/commands/` still work and support the same frontmatter as skills; the documented frontmatter fields include `model` (valid values include the same as `/model`), `description`, `name`, `allowed-tools`, `disable-model-invocation`, etc. wrap-session.md declares only `model: sonnet`, which is current-doc-compliant. No `description` field is present in wrap-session.md frontmatter — the docs note `description` is "Recommended" for skills, not required for commands; this is **not** a drift, but is worth noting that adding a one-line `description:` would future-proof against the unification deepening. Severity: clean — no Brokenness flag fires.
