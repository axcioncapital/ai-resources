# Pipeline Review — consult — 2026-06-04

## Summary

`/consult` is the operator-facing front door to the System Owner judgment agent — a thin dispatch command whose entire job is to validate input, classify question shape (general vs. change-shaped), read routing context in the change-shaped case, delegate to the `system-owner` agent, and pass the agent's voice through unmodified. Its design center is correct: the command holds zero domain judgment and zero grounding logic — all of that lives in the agent and its reference base, which is exactly where it belongs (principles.md § OP-6 — transfer the model, not the procedure; AP-3 — constraints over procedures). The command is lean, well-anchored, and shows a healthy recent change history (cycle-2 pipeline-review fixes, the C-1 return-size contract, the classifier extraction). The shape risk it carries today is concentrated at two coupling seams: the `disable-model-invocation: true` frontmatter that self-blocks Skill-tool invocation from inside an editing session, and the in-line full-read of `repo-architecture.md` (255 lines) on every change-shaped invocation — a deterministic per-call cost that telemetry has flagged repeatedly. Both are known and bounded; neither is broken. This is a sound pipeline with two leanness seams worth naming, not a fix-now problem.

## Innovations proposed

- **Pass routing context by path, not by value** — Step 3 reads `repo-architecture.md` in full and passes `ROUTING_CONTEXT` verbatim into the agent brief; the agent already has `Read` and could read the file itself from the path. Passing the path instead of the 255-line body removes a deterministic ~5K-token main-session cost on every change-shaped call (usage-log lines 281, 290 flag this exact bloat). This is a contract change between command and agent, so it must be weighed against the locked Decision 3 / Architecture Decision 4 that put the read in-line deliberately — name the conflict (persona.md § 5 rule 7) before acting. Affects: `consult.md` Step 3, `system-owner.md` Phase 2 Function B note. Risk class: structural change class (canonical command edit — `/risk-check` per risk-topology.md § 3).
- **Fold the Step 0 read-first gate into a one-line operator reminder** — the (a)/(b) gate is sound discipline but reads as a procedure the operator must self-run before invocation; the same intent could ride as a tightened preamble line. Low value at this read — the gate codifies a friction-log lesson (FL line 17, FADING-GATE verified) and removing the procedural framing risks re-opening that friction. Affects: `consult.md` Step 0. Risk class: minor.

## Leanness fixes

- **Collapse the duplicated Step 2 routing rule** — Step 2 states the change-shape routing rule inline ("If QUESTION matches… set SHAPE = change-shape and proceed to Step 3"), and the classifier doc § Consumer-specific routing rules states the identical rule for this consumer verbatim. One of the two is redundant; the consumer-specific rule belongs with the consumer (the doc says so), so the doc copy is the candidate for trimming, not the command. Net: keep the command's copy, drop the `/consult` bullet from the classifier doc's consumer-routing section. Estimated token reduction: small. Risk class: minor (doc edit, no runtime effect — risk-topology.md § 5 zero-risk zone).
- **Compress the Step 4 brief boilerplate** — the verbatim agent brief restates instructions the agent already holds in its own definition ("Apply the procedure in your agent definition: read the three references + systems-building-principles.md, apply the per-function read map…"). The agent's Phase 1–4 already mandate all of this. Trimming the restatement to the load-bearing inputs (Function letter, QUESTION, ROUTING_CONTEXT, output contract) leans the brief without losing instruction (AP-3 — the agent definition is the durable constraint home). Estimated token reduction: small. Risk class: minor.

## Brokenness / drift noted

- **`disable-model-invocation: true` self-blocks in-session Skill invocation** — Evidence: usage-log line 106 — "Mid-session add of `disable-model-invocation: true` to consult.md blocked the editing session's own Skill-tool invocation of /consult at Phase 6, causing 1 failed Skill call." This is a real, observed two-end interaction: the flag is correct for its purpose (preventing Claude from auto-firing a load-bearing advisory, matching the current platform guidance for side-effecting workflows), but it means any pipeline that tries to invoke `/consult` as a Skill (e.g., `/risk-check` Step 4a on a non-GO verdict, per usage-log line 26) must fall back to spawning `system-owner` directly via Task. The fallback works; the cost is one wasted Skill call on first encounter. Severity: Minor — known, with a working fallback, but the seam should be documented at the `/risk-check` consumer so it stops surprising editing sessions.
- **Currency check (subsumed from deprecated `/audit-critical-resources` 2026-05-29)** — WebFetch of `https://code.claude.com/docs/en/skills` succeeded. consult.md's frontmatter (`description`, `model: opus`, `disable-model-invocation: true`, `argument-hint`) maps cleanly to the current documented convention: the doc confirms `argument-hint`, `disable-model-invocation`, and `model` as supported frontmatter keys, and explicitly states "Files in `.claude/commands/` still work and support the same frontmatter" and "Custom commands have been merged into skills." No divergence — `/consult` is current under the unified slash-commands-and-skills convention. The doc notes commands could optionally migrate to the `.claude/skills/<name>/SKILL.md` shape to gain a supporting-files directory, but this is an optional feature, not a divergence; flagging it would be padding. Severity: clean.
