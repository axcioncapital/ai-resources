# Pipeline Review — context-pack-builder — 2026-06-05

## Summary

`context-pack-builder` is an interactive, gate-driven skill that reconstructs precise intent from vague operator input and emits a context pack with strict Facts/Assumptions/Unknowns epistemic separation — it is the design ancestor of the entire project-planning pipeline. Its design center is the epistemic-discipline contract (QS-4 evidence/interpretation separation) plus the "Fresh Claude Test": a pack that lets a zero-context Claude execute without re-asking. The skill is sound in isolation and has carried zero friction signal since creation. Its risk today is not internal — it is contract drift against the downstream consumer: `projects/project-planning/pipeline/ref-context-pack.md` explicitly names this skill its "structural benchmark," but the reference has since grown an 11-element schema with input-authority tags (element 9) that this skill's template does not emit. That divergence is the load-bearing finding. The currency check against the unified skills doc surfaced only Minor cosmetic frontmatter divergences.

## Innovations proposed

- **Add input-authority tags to the Inputs Available section** — The downstream `ref-context-pack.md` element 9 requires `primary-authority` / `reference` / `non-authoritative` tags per input, and those tags flow into the project plan's Canonical Inputs ranking and the harness context-loading budget (must-read / read-if-needed / do-not-read). The skill currently emits Inputs Available with no authority dimension, forcing the operator or `/plan-draft` to backfill the tags. Carrying them at pack-build time is the right home — it is where the closed evidence universe is first established (QS-4, OP-3: surface authority at source rather than letting it be silently inferred downstream). Affects: `skills/context-pack-builder/SKILL.md` (Context Pack Template, Inputs Available block). Risk class: structural change (canonical skill edit; `/improve-skill` pipeline + `/risk-check` per DR-8).
- **Promote the Evidence Table from optional to default for primary-authority packs** — The grounding-requirement evidence table is gated behind "High-Stakes Packs," but any pack that feeds `/new-project` becomes the binding intent spec for an entire project's execution. Defaulting the table on whenever a `primary-authority` input is present aligns the skill with QS-4's training-data prohibition at the point of maximum leverage. Affects: `skills/context-pack-builder/SKILL.md` (Evidence Table section). Risk class: minor (additive instruction, no contract change).

## Leanness fixes

- **Collapse the duplicated probe-question lists** — The "Probe for Missing Clarity" bullets (Epistemic Labeling section) and the "High-value questions to consider" table (Step 2) overlap on the facts-vs-assumptions and source-documents questions. One canonical question bank referenced from both points removes the duplication. Estimated token reduction: small (~150–250 tokens). Risk class: minor.
- **Single Fresh-Claude validation checklist** — The Step 4 validation checklist and the closing Quality Criteria list restate the same five tests (executable / bounded / measurable / concrete / epistemically clear) in two vocabularies. Consolidating to one and cross-referencing trims redundancy without losing the gate. Estimated token reduction: small. Risk class: minor.

## Brokenness / drift noted

- **Contract drift: skill template lags its own downstream reference schema** — `projects/project-planning/pipeline/ref-context-pack.md` (lines 5, 140–155) declares an 11-element context-pack schema, names this skill as "the structural benchmark," and adds input-authority tags (`primary-authority` / `reference` / `non-authoritative`) on element 9 that flow into the project plan and harness budget. The skill's own template (`SKILL.md` lines 66–69, Inputs Available) has no authority dimension. Evidence: `ref-context-pack.md:140-155` vs. `SKILL.md:66-69`. Severity: Substantive — the benchmark and the artifact it anchors disagree on a load-bearing field.
- **Composition gap: skill is not invoked by the pipeline that depends on its output shape** — `/plan-draft` (`projects/project-planning/.claude/commands/plan-draft.md`) consumes a `context-pack.md` and reads `implementation-project-planner/SKILL.md`, but never invokes or reads `context-pack-builder`. The skill is referenced live nowhere in `.claude` — only in audit artifacts (grep across `.claude`, `ai-resources/.claude`, `projects/project-planning`). The planning cycle (`/plan-draft` → `/plan-refine` → `/plan-evaluate`) produces the context pack `/new-project` consumes by some other path, while this skill — the documented benchmark for that artifact — sits outside the live pipeline. Evidence: grep returned only `audits/*` matches; `plan-draft.md:11-15`. Severity: Substantive — the canonical pack-builder and the pack-producing pipeline are decoupled; the schema they should share is the drift vector above.
- **Frontmatter `name` field present but redundant under unified convention** — Per the currency check (below), the directory name `context-pack-builder` is what sets the invocation; `name` in frontmatter is now a display label only. Not broken — flagged for awareness, not action. Severity: Minor.

- **`/new-project` (`.claude/commands/new-project.md`) — composition gap** — `/new-project` hard-requires `pipeline/context-pack.md` at Stage 3a First Run (`new-project.md:67-68`, exits if absent) and copies it as a discovered input. Risk: Substantive — a load-bearing artifact (`risk-topology.md § 2`: `/new-project` affects three projects) is shaped by a reference that has drifted from its benchmark skill.
- **`ref-context-pack.md` — contract drift** — The reference doc anchors to this skill as benchmark but has evolved past it (11-element schema, authority tags). Closing the drift means deciding the canonical direction per OP-11 (surface divergence, then either correct practice to the principle or revise the principle to the deliberate evolution): either the skill absorbs element 9 authority tags (the recommended direction — see Innovations), or the reference formally supersedes the skill as benchmark and the skill is marked downstream-of-reference. Silent continuation violates OP-3. Risk: Substantive.
- **`implementation-project-planner` / `implementation-spec-writer` (registry siblings) — downstream consumers** — Both are reviewed alongside this skill in the weekly tier and sit downstream of the context pack in the planning chain. No direct contract with this skill, but the authority-tag drift propagates through them (the tags feed the project plan's Canonical Inputs ranking that `implementation-project-planner` produces). Risk: advisory — fixing the source drift removes a re-derivation step downstream.
