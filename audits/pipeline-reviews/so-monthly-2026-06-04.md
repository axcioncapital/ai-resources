# Pipeline Review — so-monthly — 2026-06-04

## Summary

`/so-monthly` is the monthly macro-review command of the System Owner Friday cadence: it locates the newest `/friday-checkup` report, gates on monthly/quarterly tier, gathers the four most recent Friday advisories and the deferred-items extract from `maintenance-observations.md`, then delegates to the `system-owner` agent (Function G) which writes a strategic Monthly System Review. The design center is sound — it is a thin orchestrator that gathers inputs and hands a well-shaped brief to a judgment agent, matching the Subagent Contracts posture in `ai-resources/CLAUDE.md` and the automate-execution/gate-judgment split in `principles.md § OP-2`. The shape risk it carries today is a two-end contract defect: the command body cites the wrong grounding-map function letter (Function F) for a Function G command, the exact W21 function-letter collision that `grounding.md § 2` flags as silent mis-grounding. A second, lower drift signal is that the command file is not git-tracked at its registry path, so the cadence has no change history to review against. Neither defect is cosmetic — both touch correctness of the produced review.

## Innovations proposed

- **(none — pipeline shape is sound at this read)** — The orchestrator is correctly thin: gather inputs, gate on tier, delegate to the judgment agent, echo back. Adding pre-synthesis or extra read scope to the command body would pull judgment into the orchestrator and violate `principles.md § OP-2` (automate execution; gate judgment) and `§ AP-3` (procedural over constraint-based instructions). No load-bearing innovation is grounded by the references read.

## Leanness fixes

- **(none — pipeline is already lean)** — At ~65 lines the command carries no redundant validation, no prose padding, and no gate firing on an already-enforced upstream condition. The tier gate (Step 1) and the missing-report abort are the only guards, and each guards a distinct failure. Trimming would remove correctness, not fat.

## Brokenness / drift noted

- **Function-letter collision: command cites Function F, but `/so-monthly` is Function G.** Evidence: `ai-resources/.claude/commands/so-monthly.md:59` — "The agent reads its references and vault docs per grounding.md Function F read map." `grounding.md § 3 Step 1(g)` and `§ 2 Function G` define `/so-monthly` as Function G; `§ 2` opening note names this exact defect: "A command citing the wrong letter silently mis-grounds the agent; this is the defect class behind the W21 function-letter collision (Decision D-5/D-6)." Function F (friday advisory) reads `principles.md` + `system-doc.md` by default; Function G reads `principles.md` + `blueprint.md` + `system-doc.md` + `repo-state.md`. A command-driven Function F read would drop `blueprint.md` and `repo-state.md` — the two docs the monthly review's "Strategic Observations" and state-aware "Patterns This Month" sections depend on. The agent's own Phase 2 self-detects function shape from the `function: monthly-review` brief field (`so-monthly.md:37`, agent `system-owner.md:49`), so the live blast radius is partially contained — but the command body is wrong on its face and violates `grounding.md § 2`'s command-body-consistency rule. Severity: Substantive.

- **Same collision present in the sibling `/friday-so` (off-by-one pattern, not isolated).** Evidence: `ai-resources/.claude/commands/friday-so.md:52` cites "Function E read map," but `/friday-so` is Function F per `grounding.md § 3 Step 1(f)`. `/systems-review` (the true Function E) cites Function E correctly (`systems-review.md:47,59,67`). Severity: Substantive (names a second live command).

## Cross-resource interactions

- **`/friday-so` — contract drift (shared concern: grounding function-letter citation)** — Both commands delegate to the same `system-owner` agent and both mis-cite their `grounding.md` function letter (see Brokenness bullet 2). The two pipelines disagree with `grounding.md § 2/§ 3` on a shared concern — the function→command mapping — in the same off-by-one direction. Fixing `/so-monthly` without fixing `/friday-so` leaves the pattern half-corrected. Risk: Substantive.

- **`/friday-checkup` — upstream/downstream dependency** — `/so-monthly` is the terminal consumer of the `/friday-checkup` report: Step 1 globs `friday-checkup-*.md`, reads the newest, and aborts if the tier is weekly. `friday-checkup.md` is Critical in `risk-topology.md § 1` ("Orchestrates entire maintenance cadence"). The `**Tier:**` line and the `friday-checkup-*.md` filename pattern are two-end contracts per `risk-topology.md § 3` ("source-label strings are two-end contracts"). A change to the checkup report's tier-line format or filename would silently break `/so-monthly`'s gate. No drift detected today — flagged as a standing coupling to absorb if `/friday-checkup` changes. Risk: advisory.

- **`system-owner` agent (Function G output shape) — composition handoff, contracts match** — The command's brief (`so-monthly.md:37–56`) supplies `function: monthly-review`, the checkup report, N advisories, deferred items, and `Output path`. The agent's Function G section (`system-owner.md:131–140`) consumes exactly these and writes to `output/monthly-reviews/so-monthly-{DATE}.md`, matching the command's `OUTPUT_PATH` (`so-monthly.md:29`). Step 6 echoes the `## System Health Summary` section; the agent produces that heading. The handoff contract is aligned. One minor seam: Step 2 globs `friday-advisory-*.md`, which matches versioned duplicates like `friday-advisory-2026-05-08-v2.md` (present in the output dir) — the "4 most recent" could include a v1+v2 pair of the same advisory, double-counting one week. Risk: advisory.
