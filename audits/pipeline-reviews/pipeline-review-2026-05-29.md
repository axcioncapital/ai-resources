# Pipeline Review — pipeline-review — 2026-05-29

## Summary

`/pipeline-review` is the weekly Sonnet orchestrator that drives the registry-shortlist → operator-pick → per-pipeline Opus auditor fan-out → batched registry-bump loop. Its design center is two-tier registry-driven cadence (32 weekly + 15 quarterly) plus subsumption of the deprecated `/audit-critical-resources` currency-check into the auditor's Brokenness section — one cadence, two jobs (innovation review and drift detection). The pipeline carries three live risk shapes today: (i) self-reference fragility — this command writes the registry it reads, so a partial-batch write or a same-day re-run can corrupt audit-trail-grade history (`risk-topology.md § 1 Critical`, since `/pipeline-review` already appears in the Critical row alongside `friday-checkup.md`); (ii) the just-relaxed pick cap removed the cost ceiling without adding any throughput governor downstream — the operator's fix-session queue is now the implicit gate, undocumented in the pipeline body; (iii) frontmatter has only `model: sonnet` — no `description`, no `disable-model-invocation`, no `argument-hint`, which diverges from the unified Anthropic skills/commands convention shipped on `code.claude.com/docs/en/skills`. The shape is sound; the gaps are correctable inline.

## Brokenness / drift noted

- **Frontmatter divergence from unified skills/commands convention** — `pipeline-review.md` declares only `model: sonnet`. The current unified Anthropic doc (`code.claude.com/docs/en/skills`, fetched 2026-05-29) names `description` as the field that surfaces a command/skill to Claude's invocation engine, and `disable-model-invocation: true` for operator-only workflows. `/pipeline-review` is operator-only by design (Section 3 of `persona.md`: every consultation is operator-invoked, and this command writes audit-trail-grade rows that must not auto-fire). Evidence: pipeline-review.md lines 1–3 vs. unified docs lines 192–217 (frontmatter reference table). Severity: Substantive — without `disable-model-invocation: true` the command is in scope for Claude-initiated invocation, which violates the audit-trail-grade write contract (registry bump without operator pick). Add: `description:` (one sentence) + `disable-model-invocation: true` + optional `argument-hint: [pipeline-path]`.
- **Same frontmatter gap on the auditor** — `pipeline-review-auditor.md` declares `name`, `description`, `model: opus`, `tools` correctly. The subagent contract is fine. No fix needed here; the gap is on the orchestrator only.
- **Step 4.17 numbering collision** — `pipeline-review.md` has two Step 4 sub-steps numbered 17 (line 79 "Parse `REGISTRY_PATH`" and line 80 "Tier-eligibility filter" labeled `17.1`, then `17.5`). Severity: Minor — purely cosmetic; the step body executes correctly. Renumber 17.1 → 17.5 → 18a/18b/18c when next touched.
- **Currency check — `command`-type pin (against `code.claude.com/docs/en/skills`)** — WebFetched 2026-05-29. The unified docs explicitly state: "Custom commands have been merged into skills. Two divergences from current platform guidance: (i) the `description` field is the primary metadata, and the pipeline lacks it (Substantive — see frontmatter bullet above); (ii) the doc recommends skills over commands for new work, which is forward-looking advice; `/pipeline-review` is a multi-step orchestrator and stays correctly in `.claude/commands/` per `principles.md § DR-3`.
