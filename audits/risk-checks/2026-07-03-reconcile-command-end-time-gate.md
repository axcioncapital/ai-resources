### Dimension 5: Hidden Coupling
**Risk:** Medium

This is where the executed change's residual risk concentrates, and it is the operator's flagged question ("the nested Skill-tool delegation pattern, zero prior precedent — does it introduce risk beyond what `/qc-pass` already caught?"). Answer: yes, a bounded residual — `/qc-pass` fixed the "delegation can't fire" blocker; the residual is the opposite tail, "it fires but silently degrades," across four implicit dependencies. All four degrade cleanly and are documented at the change site, which is why this is Medium and not High.

- **1. Novel `Skill(contract-check)` delegation with a logged prior failure mode.** `Skill(...)` appears nowhere else in the repo as a frontmatter grant (grep-confirmed), so this is genuinely first-of-kind. And the mechanism has a recorded failure precedent: `ai-resources/logs/usage-log.md:16` — "One abandoned project-consultant spawn when `Skill("consult")` mis-resolved (/consult not surfaced as a Skill this session)." So Skill-tool-invokes-a-command **is known to sometimes not resolve**. Mitigant: reconcile.md Step 3 (lines 48–49) explicitly handles abort/error/mismatch by setting `CONTRACT_CHECK_RESULT = "unavailable — {reason}"` and proceeding; reconcile-reviewer treats the cross-check as "corroborating evidence only, never a hard dependency" (line 49). The pattern therefore **cannot silently break `/reconcile`** — worst case it loses one corroborating signal.
- **2. `/contract-check` is not currently reachable in the project.** `ls projects/buy-side-service-plan/.claude/commands/` shows only `qc-pass.md` (symlink) and `reconcile.md` (regular file) — **no `contract-check.md`**. It would be auto-symlinked on the next SessionStart (it is not in `EXCLUDE_COMMANDS` or the manifest `.local` list), but until that hook runs — and if the Skill tool still fails to surface it — the delegation is a no-op that degrades to "unavailable" per finding 1. Implicit dependency, documented-at-site.
- **3. Output-string parse-coupling (the sharpest edge).** reconcile.md Step 3 (line 47) parses `/contract-check`'s Step 5 human-readable echo `Artifact: '{path}' (auto-detected from uncommitted/today's changes).` (contract-check.md line 103) to decide whether the cross-check targeted the right artifact. This is an **undocumented reverse contract**: reconcile hard-codes a dependency on another command's prose output line, and `contract-check.md` has no knowledge that its wording is now a parse target. If contract-check's echo string is reworded, reconcile's parse fails — but, again, it fails *closed* into "unavailable," not into a wrong verdict.
- **4. Forensic-source convention reliance.** reconcile-reviewer reads `session-notes.md`/`decisions.md`/`git log` and assumes their conventions. Both files exist for this project; reconcile-reviewer Step 1 (line 35) documents the fallback ("If none of the forensic sources cover the target output's production, say so explicitly … rather than inventing"). One implicit dependency on an established repo convention, handled gracefully.
- **The plan-time nested-subagent worry is resolved in the executed code.** The plan-time gate flagged that a subagent cannot spawn `/contract-check`'s own `Task` subagent, so the `/contract-check` call had to sit in the *main* command, not inside `reconcile-reviewer`. reconcile.md does exactly this: Step 3 fires `/contract-check` and Step 4 then spawns reconcile-reviewer, passing `CONTRACT_CHECK_RESULT` in as a plain input (lines 45–59). Correctly pinned — not a residual risk.
- **Naming adjacency persists but is ergonomic-only.** `/reconcile-backlog` exists as a shared command (`ai-resources/.claude/commands/reconcile-backlog.md`) and will auto-sync into buy-side-service-plan, so that project will carry two `reconcile*` commands with different concerns. Distinct filenames → no dispatch collision; this is operator mental-model friction, not a functional break.
- **Why Medium, not High:** there *are* multiple implicit dependencies (findings 1–4), which is the literal High trigger — but every one is **non-load-bearing** (the `/contract-check` cross-check is corroborating-only by design) and **fails closed** to "unavailable/proceed," and each is documented at the reconcile.md change site with a named fallback. No dependency can silently corrupt a verdict. **Why not Low:** the delegation is a genuinely novel pattern with a logged prior failure mode, and reconcile parses another command's prose output as an undocumented contract.
- *Optional hardening (advisory, not required for GO):* add a one-line comment in `contract-check.md` Step 5 noting that its `Artifact:` echo is parsed by `/reconcile`, so a future reword is done knowingly (converts the finding-3 reverse contract from implicit to documented). Non-blocking.
