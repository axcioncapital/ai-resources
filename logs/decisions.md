## 2026-07-14 (S1) — Permission-class re-cut: arbitrate the authority split, separate the axes, and gate the chassis version

**Context.** Mission `research-workflow-deploy-fitness`, thread 5 ("clarify the evidence rules"). The thread's stated defect — a hole in the class thresholds ("SUPPORTED needs ≥3 sources/≥2 classes") — **does not exist**: those thresholds appear in no file (canonical chassis, `claim-permission.template.md`, and both live projects' `quality-standards.md` all grepped). Third consecutive thread whose audit-stated premise collapsed on a direct read. The area was broken anyway, differently.

**Decision 1 — the Conditions column is CHASSIS-OWNED; class names are canonical-global.** The claimed self-contradiction (the skill hard-codes the four class names while its Behavior step calls the project's `quality-standards.md` authoritative) was **not one**. The chassis's Canonical-ordering rule already fixes class *names* globally and makes only *thresholds* project-fillable — so hard-coding the names is correct. What was genuinely undefined, and load-bearing, was the **status of the Conditions column**: neither declared canonical nor declared project-fillable. Settled explicitly: names + verb lists + conditions + ceilings are chassis-owned; per-claim-type thresholds are project-fillable calibration *on top of* the generic bar. **Alternative rejected:** making the Output schema parse class names dynamically from the project file. That would make the class vocabulary project-variable, breaking every downstream exact-string match, to serve a flexibility no project has ever used (both deployed projects carry byte-identical class names).

**Decision 2 — separate the axes; do not add a class.** The mandate forbade a fifth class, and none was needed. The table was doing two jobs in one column: grading the **evidence** and bounding the **claim's ambition**. Split them — the class grades evidence on one ordered axis (independent evidentiary roles → fit), and the instance-count / country-coverage conditions become **ceilings** applied afterward, reusing the already-risk-checked risk-tier ceiling mechanism. This also, for the first time, gives the orphan `illustrative`/`directional`/`pattern-candidate` ladder a home: it is a **claim-scope** ladder, orthogonal to the class. **Alternative rejected:** adding a "single-source" class to absorb the gap — it would have papered over the mixed-axis root cause and left the overlap untouched.

**Decision 3 — an evidenced negative is a first-class finding; `NOT-SUPPORTED` never means "false".** Evidence that positively supports a negative conclusion (a register recording a true zero) grades on the ordinary ladder and **can reach `SUPPORTED`**. A failed search is `NOT-SUPPORTED` — a statement about the evidence, not about the world — and licenses **neither the claim nor its negation**. Incomplete research is a *process* penalty and must be recorded as one, never as an evidence verdict. This required **new Stop Condition 5** (max passes exhausted): Conditions 1–4 could never be satisfied by a subtask that found exactly one source, so the reciprocal rule downgraded every such claim to `NOT-SUPPORTED`, making `ILLUSTRATIVE-ONLY` **structurally unreachable**. The class survived only because no skill ever implemented the reciprocal rule — an unenforced rule masking a contradiction, not a safeguard.

**Decision 4 — accept `/risk-check`'s RECONSIDER; ship a chassis-version hard-exit gate.** The reviewer caught a hazard I had missed entirely: the canonical **skills are symlinked** into both live projects, but each project holds a **real local copy of the chassis**. A merge updates the *consumers* and leaves the *rules* stale — and every existing pre-flight check is a *heading-presence* check that an old chassis passes, so nothing could detect it. It would have **silently misadjudicated**. Closed by adding a chassis-version marker + a pre-flight gate in both skills that **hard-exits** on a stale chassis. **Alternative rejected:** a warning instead of a hard exit. Silent misadjudication of evidence claims is the worst failure this gate can have; a loud halt is the correct trade, and a warning would be ignored exactly as the old "no `{{` anywhere" assertion was (the thread-2 pathology).

**Decision 5 — defer the back-port; do not widen scope to fix it.** Neither live project's chassis was back-ported. This is an explicit, operator-approved mandate scope bound ("canonical only") and a declared stop condition. Deferred to the operator, logged as an OPEN action in **both projects' own `logs/decisions.md`** (so it lives where those projects' operators will see it), and mechanically protected by the hard-exit gate. Exposure is low: both projects carry `.claim-permission-gate.done` sentinels for §1.1, so the gate does not re-run today. **Required before either project processes a new section or anyone deletes a sentinel.**

**Decision 6 — correct my own overstatement rather than let it stand.** I had logged that permission enforcement happens "nowhere". The `/risk-check` consumer inventory showed `section-directive-drafter` **is** a real live consumer (it converts classes into per-finding prose constraints). Narrowed the finding to the accurate claim: classes ARE converted to prose constraints at Stage 3, and are enforced by **nothing** at Stage 4.3. A materially smaller (and differently-shaped) hole than "enforced nowhere".

**Verification method (the thing that made all of this land).** Verified **by execution, not by reading** — the mission's standing rule after threads 1 and 2. Four blind adjudication runs: fresh Opus subagents dispatched *as* the skill, against a 5-claim adversarial fixture, never told the expected answers. **OLD rules: `GAPS 1, OVERLAPS 1`** — the agent independently diagnosed the mixed-axis root cause unprompted ("channels and instances are different axes; the class table treats them as if they were one"). **NEW rules (final): `GAPS 0, OVERLAPS 0, UNDETERMINED 0`.** Three correction rounds in between; **each run found real defects in the previous fix, two of them mine.**

**Decided by:** Claude (S1). Premise correction, the deferred back-port, and the strategic finding below were all stated to the operator before commit.

**Strategic consequence, stated and NOT resolved.** The mission's premise ("fix canonical before deploying") has now produced **zero demonstrated blockers across three threads** (1, 2, 5). This is not chance: the source audit reasons from **what the files say, not what the runtime does**, and every time execution has been applied its conclusion has flipped. Threads 3, 4, 6, 7, 8 come from the same audit by the same method. **Recommendation on record: re-examine the deployment gate before spending another session inside the mission.**

## 2026-07-14 (S3) — Outputs-side chassis-provenance gate: stamp verdicts with their rules, and route forgery to the existing override rather than build a hash

**Context.** Thread 5 (prior session, S1) shipped a chassis-version gate that stops a *new* run against *old* rules. It said nothing about permission-table verdicts already on disk. Tested by execution: re-adjudicating a real stale table (research-pe, 1.1-cluster-03) under the current chassis moved 2 of 6 claims to a stricter class, and a blind `section-directive-drafter` run on the untouched stale table confirmed it would license a market-pattern generalization the current rules forbid. An over-claim was being laundered into two live projects' reports with nothing in the pipeline able to see it.

**Decision.** Both producers (`claim-permission-gate`, `cluster-memo-refiner`) now stamp `chassis_version:` into permission-table frontmatter; three consumers (`section-directive-drafter`, `/run-synthesis`, `/run-sufficiency` Step 0) hard-exit on a missing, pre-2026-07-14, or re-stamped (`generated_at` < `chassis_version`) value. `country-parity-checker` was deliberately left ungated — verified it never reads the Assigned-class column, so a stale table cannot corrupt its output.

**No content hash was built.** An adversarial test proved the two self-asserted frontmatter fields can be forged with two hand-edits. A hash would close that gap. Rejected because these are model-executed skills, not scripts — they cannot compute a trustworthy digest without new shell machinery — and the actual threat is a documented operator hand-edit path (`/run-sufficiency` § Operator-override), not an adversary. Operators who genuinely mean to proceed on a stale table are routed to the existing signed `OPERATOR-OVERRIDE` path (logged to this file) instead. The gate's limit is stated plainly in the chassis rather than oversold.

**Alternatives considered:** (a) a content hash over the verdict rows — rejected, above; (b) gating `country-parity-checker` too, for uniformity — rejected, no hazard exists there and it would be ceremony; (c) leaving the first-draft refusal message ("delete the sentinel and re-run") as-is — rejected after `/risk-check` showed it dead-ends an operator in a second hard exit, since `claim-permission-gate` carries its own chassis gate and both live projects have unversioned chassis copies; every refusal message now states back-port-first ordering explicitly.

**Verification.** Tested red/green/adversarial before AND after the fix: unversioned table → EXIT; forged stamp (version pasted, body untouched) → PROCEEDED (the defect); re-stamp invariant added → forged stamp now EXITS, genuine table still PROCEEDS. `/risk-check` verdict PROCEED-WITH-CAUTION; all five required mitigations applied before commit, including a third consumer (`/run-sufficiency` Phases D+F) the reviewer found and my own inventory had missed twice.

**Decided by:** Claude (S3), on the mandate's own stop condition to test the premise before fixing it. Stated to the operator before commit and again before merge.

## 2026-07-14 (S5) — Treat "delete the marker, then re-run the destructive command" as a DEFECT to be logged, not a workaround to be used quietly

**Context.** Closing the `ai-resources-research-workflow` worktree, `check-destructive-liveness.sh` hard-blocked `git worktree remove` on the evidence of a live session marker (`2026-07-14 S3`). The worktree was in fact idle — its S3 session had run `/wrap-session` to completion and committed at 12:11:37 (`ca68eaa`). The marker was a **corpse**: the `SessionEnd` teardown had not fired. The only mechanical way forward was to delete the marker files **the guard reads** and re-issue the blocked command.

**Decision.** Do it — but (a) **state in chat, before acting, that this is clearing false state and not working around the guard**, (b) get the operator's explicit liveness confirmation first (the hook's own contract: *"liveness is the one fact only the operator holds"*), and (c) **log the bypass itself as a HIGH defect** rather than banking it as a technique.

**Rationale.** The corpse converts a *fail-closed safety guard* into a *routine obstacle*, and routine obstacles get routinely bypassed. The recipe "erase the evidence, retry" now demonstrably works, and it was discovered in the course of **legitimately** using the guard — which is exactly how such recipes spread. The next session meeting this block under time pressure will have a precedent for erasing state rather than establishing liveness. An override that is **indistinguishable from tampering cannot be audited**; the hook therefore needs an explicit, logged operator-override input, not one that functions by deleting what it reads.

**This also falsified a standing hypothesis.** The open question since S13 — *why do finished sessions leave marker corpses?* — assumed **crashed** sessions. False: S3 wrapped cleanly and still left both markers. So the guard false-fires on **every** finished session, and `close-worktree-session.md` L127–131 (which asserts the corpse problem was fixed on 2026-07-13, and warns that otherwise "this check would have false-fired on every already-finished session") is **false as written** — that is precisely the state we are in.

**Alternatives considered.** (a) *Delete the markers silently and proceed* — rejected: it is the same keystrokes as tampering, leaves no record, and would have buried the strongest evidence yet for a HIGH open item. (b) *Refuse to proceed at all* — rejected: the worktree was genuinely idle, the operator confirmed it, and the hook's documented "known gap" clause explicitly makes this the operator's call once they say so. (c) *"Fix" it by pruning stale markers as routine wrap hygiene* — rejected, and explicitly recorded as a **do-not** in the improvement-log entry: it treats the symptom and re-arms the bypass.

**Sequencing note (separate, and the reason nothing was lost).** The branch held **one commit reachable from nowhere else** (`ca68eaa`, 122 lines). Merged to `main` **before** any removal, then deleted with `git branch -d` (lowercase), so **git certified** the branch was fully merged rather than me asserting it. `-D` would have destroyed the commit silently against a perfectly clean `git status`.

**Decided by:** Claude (S5), within the operator's instruction to close the worktree. Operator supplied the liveness confirmation the guard requires and directed that the problems be logged.

## 2026-07-14 (S7) — The pilot's enforcement half was cut on its own gates' verdicts; the diagnostic half shipped and was proven by regression test

**Context.** The operator asked to implement the approved repo-repair pilot V1. The plan explicitly gates its own construction — it edits `protected-zones.md` and `audit-discipline.md`, architectural tier under the very rules it installs — and states the test: *"If the plan cannot pass the gate it installs, the gate does not work."* Both plan-time gates were run.

**Decision.** **Land Half 1 (the two reviewer edits + the three `risk-check.md` consumer fixes). Do NOT land Half 2 (the `require-gate.sh` blocking hook, its `settings.json` wiring, the `Files-checked:` footers, the two doctrine-doc edits, the `warn-settings-change.sh` deletion).** `/consult` → **CUT BACK**; `/risk-check` → **RECONSIDER** (High on usage cost, blast radius, reversibility, hidden coupling).

**Rationale.** Three facts, each verified by execution or direct read, kill the hook *as specified*:
1. **`warn-settings-change.sh` — the file the plan calls *"proof the `exit 2` pattern works"* — FAILS OPEN.** Fed a real `PreToolUse` payload it exits **0**. It reads a top-level `file_path`; the payload nests it under `tool_input`. Every functioning hook in the repo reads `.tool_input.file_path`; it is the only one that gets the contract wrong, **and the only one nobody ever wired — which is why the bug survived. An unwired hook is never observed to fail.** Copying it ships a guard that blocks nothing: the repo's most-repeated failure mode ("inert safeguard", 6 logged instances), reproduced by the plan written to end it.
2. **It would not fire where it matters.** Wired in `ai-resources/.claude/settings.json`, it does not load for the ~20 project-rooted sessions that edit `ai-resources` files via `--add-dir`. Workspace-root `.claude/settings.json` has **no `PreToolUse` block at all**. The hook's own stated purpose — *"enforces the gate at the file, regardless of which door you came through"* — is false for most doors.
3. **Self-lock with a self-defeating escape.** Once wired it blocks `Edit`/`Write` to itself and to the `settings.json` it is wired in. The only recovery is the Bash path (`sed -i`, `cat >`), which is simultaneously why the gate enforces nothing (`Bash(*)` + `bypassPermissions`).

**The doctrine edits were held back with the hook, deliberately.** They are the *policy*; the hook is the *enforcement*. Landing `protected-zones.md`'s waiver deletion alone makes `/risk-check` mandatory on every command/agent edit **with no mechanism to enforce it** — precisely the "usage cost: High" the gate flagged, and it yields a rule one must *remember* to obey. The plan's own thesis: *"a rule you must remember to read is not a control; it is a wish."* Cost without protection is worse than neither.

**Verification — the plan's own stop condition, executed.** REGRESSION TEST A: the fixed `qc-reviewer`, dispatched **blind** (never told the expected answer) against `audits/research-workflow-deployment-fitness-2026-07-13.md` — the audit the **old** agent passed with a **GO**, whose findings were mostly wrong. **Old: GO → New: REVISE.** It caught **F-1's invented consequence** by *tracing it to the caller* (`run-sufficiency.md:57` passes the memo directory at dispatch, so the *"unconditional deadlock… this alone forces the verdict"* is false), and it caught **F-9 contradicting F-13(b)** inside the same document. It then found four more, including that the audit's **entire cited evidence base does not exist**. Red before, green after — not a code read.

**Alternatives considered.** (a) *Land the whole plan as approved* — rejected: `audit-discipline.md` verdict semantics forbid downgrading RECONSIDER to push a change through, and the hook's template provably fails open. (b) *Land the doctrine edits without the hook* — rejected, see above. (c) *Land nothing and defer wholesale* — rejected: Half 1 is independently gated-clear, aimed at the verified root cause, reversible, and has a true regression test with a known answer. Deferring it would have deferred the only part that was proven to work.

**Decided by:** Claude (S7); operator confirmed with "go" after being shown the split and the three verified facts.

## 2026-07-14 (S7) — A gate that grades an artifact cannot catch a false premise; the premise check outranks the other dimensions rather than averaging with them

**Context.** Every gate in this system grades an **artifact** (is the change safe, well-scoped, well-formed?). **Not one grades a claim.** So a false premise passes every gate and each one adds authority to it — which is how `/consult` once reasoned impeccably about a bug that did not exist, and how an audit that was 7-of-8 wrong collected a `GO` from `qc-reviewer` (whose citations were all *accurate*; the invented part was the **consequence**).

**Decision.** Add **Dimension 7 — Problem Reality** to `risk-check-reviewer` and a **premise check** to `qc-reviewer` — and make both **outrank** the other dimensions rather than average with them. A `High`/`INCOMPLETE` on Problem Reality forces `RECONSIDER` on its own; a claim whose consequence is untraced or contradicted forces `REVISE` regardless of how clean dimensions 1–6 are. Both grade the **defect** and its **consequence** *separately*.

**Rationale.** Averaging was the defect. Six Lows outvoting one false premise is *exactly* the arithmetic that produced the original `GO`. A false premise is not one risk among seven — it invalidates everything built on it, so it cannot be traded off against the other six. This also required carving out `risk-check-reviewer`'s standing instruction to *"treat the passed inputs as the entire world"*: that sentence governs **scope**, not **truth**, and left uncarved it makes the agent accept the caller's defect claim **by design** — Dimension 7 would have been dead on arrival.

**A live demonstration, same session, worth preserving.** Audit F-13(d) claimed *"three **real template-local** commands (`consult`, `run-sufficiency`, `session-plan`)"*. **The audit** said all three are real — **false**. **The QC reviewer** said two don't exist — **imprecise** (they exist as *symlinks*). **I** then said the reviewer was wrong because all three exist — **also imprecise**, because my test used `[ -f ]`, which **follows symlinks** and cannot see the distinction. Only `[ -L ]` settled it. Three parties, three errors, one insufficiently sharp test — and **the reviewer was right in substance while I was wrong.** The lesson is not "trust the checker"; it is that *nothing here that asserts without running a sharp enough command is reliable — including the checker, and including me checking the checker.* Only command output ever settled a dispute this session.

**Alternatives considered.** (a) *Score Dimension 7 as an ordinary seventh dimension* — rejected: it reproduces the averaging defect it exists to fix. (b) *A citation rule ("cite `path:line`")* — rejected, and this is the sharp point: the failing audit's citations were **all accurate**. A citation rule grades the claim; it does not grade the **inference from** the claim. Both must be graded, separately. (c) *Give `qc-reviewer` a Bash tool so it can execute its verification* — deferred, not rejected: it is a real permissions-surface expansion on an agent invoked everywhere. The current wording requires it to mark a consequence `UNVERIFIED — requires execution` and downgrade the claim rather than assert what it could not check.

**Decided by:** Claude (S7), within the operator-approved plan's design; the outranking rule and the "entire world" carve-out are S7 additions the plan did not specify.
