The user wants to add or update a rule in the project-level CLAUDE.md. Their instruction follows this prompt.

1. Read CLAUDE.md in the current working directory and understand its section structure.
2. Determine which existing section the instruction best fits in. If none fits, propose a new section name and placement.
3. Draft the new entry using the same formatting style as that section.
4. If it contradicts or updates an existing entry, replace the old one.
5. Show the proposed change (with surrounding context) and wait for approval before writing.
